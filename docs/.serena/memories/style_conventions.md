# 风格与约定

- 语言: 默认中文；命令、路径、代码标识保持原文形式。
- 结构: 文档建议采用“目的/范围 → 术语与口径 → 业务流程 → 指标与图表 → 交互与状态 → 权限与安全 → 性能与非功能 → 埋点与验收 → 风险与边界 → 版本与变更”十段式。
- 指标: 必须定义“名称、业务含义、计算口径、时间边界、主键/去重规则、过滤条件（含排除）、示例 SQL/逻辑、数据源与刷新频率、维度拆解与可用筛选、质量校验”。
- 引用: 跨文档引用时注明来源文件名与小节标题，避免同义多词（统一术语表）。
- 变更: 每次修改更新“文档版本、变更摘要、影响面、回滚/兼容策略、验证清单”。
- PowerShell: rg 命令使用外单引号包裹、内双引号包裹搜索词与路径；复杂替换使用 Here-String 或 .ps1 分步。
- 编码: 必须为 UTF-8（无 BOM）。
