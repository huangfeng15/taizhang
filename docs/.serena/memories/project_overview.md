# 项目概览

- 项目类型: 以 Markdown 为主的业务与技术文档库（采购/成本、台账、页面方案、系统架构、模板与实施指南）。
- 目标读者: 业务负责人、产品经理、数据与前端开发、测试与运维。
- 主要内容: 各业务页面需求方案（如“工作周期监控页面”）、系统与数据口径说明、实现规范、优化清单、使用手册等。
- 运行环境: Windows（PowerShell 作为主要终端）。
- 技术栈: 文档为 Markdown，暂无代码工程；Serena 以最小 project.yml 激活后用于搜索与编辑文档。
- 编码规范: 文件统一使用 UTF-8（无 BOM），禁止 GBK/ANSI；中文内容为主，命令与标识按原文。
- 约定原则: KISS / YAGNI / SOLID / DRY；文档需可执行、可落地，强调指标口径一致、术语统一、最小可用方案（MVP）。
- 目录特征: docs 目录内包含多个中文命名的 .md 文档（页面方案、架构、模板、指引等）。
