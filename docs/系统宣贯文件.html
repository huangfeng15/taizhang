1. 系统开发背景与价值

1.1 为什么要开发这个系统？

在系统开发之前，我们的采购与成本管理面临以下核心痛点：

痛点问题 具体表现 影响
数据分散混乱 多个Excel文件，版本不统一 查找困难，数据不一致
重复录入工作量大 相同信息在多个表格重复填写 效率低下，容易出错
缺乏关联关系 项目、采购、合同、付款数据孤立 无法形成完整业务链条
统计分析困难 手工汇总统计，耗时耗力 决策滞后，容易遗漏
数据质量难保证 格式不统一，校验困难 数据错误影响决策
历史追溯困难 缺乏完整的操作记录 问题难以分析和改进

1.2 系统化管理的价值

🚀 提高工作效率

  • 减少重复录入：通过外键关联，一次录入多处复用
  • 统计报表自动化：自动生成各类统计报表，减少手工统计工作
  • 减少重复性工作：自动计算、自动关联、自动校验

✅ 减少错误和不统一

  • 统一数据标准：编码规范、格式统一、字段标准
  • 自动数据校验：必填字段检查、逻辑关系验证、格式规范检查
  • 版本控制：单一数据源，避免多版本混乱

🔗 减少重复信息录入

  • 外键关联机制：项目信息自动关联到采购、合同
  • 数据继承：补充协议自动继承主合同信息
  • 智能填充：基于关联关系自动填充相关信息

🏢 统一管理平台

  • 集中化数据存储：所有数据统一管理，避免分散
  • 标准化业务流程：建立规范的业务管理流程
  • 权限分级管理：不同角色不同权限，确保数据安全

💡 挖掘数据价值，让台账发挥真正价值

传统Excel台账只是数据的记录工具，而我们的系统要让台账成为业务指导工具：

  • 从"记录"到"洞察"：实时监控、趋势分析、异常预警、绩效评估
  • 从"被动"到"主动"：主动管理、精准决策、持续改进、价值创造

2. 系统运行环境与安全

2.1 系统运行方式

🖥️ 部署架构

  • 运行环境：系统部署在本地电脑，每天开机自动运行
  • 访问方式：仅支持局域网内访问，确保数据安全
  • 服务地址：https://系统IP地址:3500
  • 访问协议：HTTPS加密传输，保障数据传输安全

📱 访问要求

  • 网络要求：必须连接公司局域网
  • 浏览器要求：推荐使用Chrome、Edge等现代浏览器
  • 系统兼容：支持Windows、Mac、Linux等操作系统

2.2 安全管理机制

🔐 登录安全

  • 登录有效期：12小时自动过期，需要重新登录
  • 会话管理：超时自动退出，防止未授权访问
  • 密码保护：定期更换密码，确保账户安全

🛡️ 数据安全

  • 权限控制：暂未分配细粒度权限，所有用户权限相同
  • 操作限制：禁止进行删除数据库操作，确保数据安全
  • 审计日志：记录所有操作行为，支持问题追溯

💾 数据备份

  • 自动备份：系统每日自动进行数据备份
  • 手动备份：支持手动导出数据备份
  • 备份位置：备份文件存储到共享盘指定位置
  • 备份策略：保留最近30天的备份文件

2.3 使用注意事项

⚠️ 重要提醒

  • 仅限局域网访问：外网无法访问系统
  • 12小时登录有效期：超时需要重新登录
  • 禁止删除操作：没有经过沟通切勿尝试删除数据库数据，因为现有数据可能已做了一些微小调整
  • 及时备份数据：重要操作前建议手动备份

3. 系统模块功能介绍

3.1 核心业务模块

🏗️ 项目管理模块

功能定位：项目的顶层组织单元，为所有业务数据提供项目维度管理

主要功能：

  • 项目基础信息管理（编码、名称、负责人、状态）
  • 项目关联数据总览（采购、合同、付款汇总）
  • 项目进展状态跟踪

🛒 采购管理模块

功能定位：记录从需求提出到中标结果的完整采购过程

主要功能：

  • 采购全流程记录（需求→公告→开标→中标→归档）
  • 采购方式管理（支持9种标准采购方式）
  • 预算控制分析（预算、控制价、中标价对比）
  • 采购周期监控

📄 合同管理模块

功能定位：管理主合同及补充协议，是付款和结算的基础

主要功能：

  • 合同全生命周期管理
  • 补充协议关联管理
  • 合同金额和付款比例计算
  • 合同归档状态跟踪

💰 付款管理模块

功能定位：记录每一笔付款交易，跟踪资金执行情况

主要功能：

  • 付款记录管理
  • 付款进度跟踪
  • 结算标记管理
  • 付款编号自动生成

⭐ 供应商履约评价模块

功能定位：记录供应商履约表现，为供应商管理提供数据支撑

主要功能：

  • 履约评价记录
  • 多维度评分管理
  • 历史记录查询
  • 评价趋势分析

3.2 数据导入导出功能

📥 数据导入

导入方式：

  • 页面导入：在各模块页面点击"导入Excel"按钮
  • 模板下载：系统提供标准导入模板
  • 数据校验：导入时自动进行数据格式和业务逻辑校验
  • 错误反馈：详细的错误信息，便于修正后重新导入

导入限定规则：

  • 必填字段检查：关键字段不能为空
  • 格式验证：日期、金额、编码格式必须正确
  • 业务逻辑验证：关联关系、数值范围、逻辑一致性
  • 重复性检查：避免重复数据导入

📤 数据导出

导出方式：

  • 页面导出：在各列表页面点击"导出Excel"按钮
  • 筛选导出：支持按筛选条件导出特定数据
  • 全量导出：支持导出全部数据（大数据量时异步处理）
  • 备份导出：定期导出数据进行备份

导出格式：

  • Excel格式：.xlsx格式，便于后续处理
  • 包含字段：完整的业务字段和关联信息
  • 格式规范：标准化的日期、金额、编码格式

4. 系统指导工作应用

4.1 指导及时更新台账数据

📅 数据更新及时性要求

更新频率要求：

  • 日常数据：业务完成后24小时内更新
  • 重要节点：关键节点完成后立即更新
  • 定期检查：每周检查数据更新情况

4.2 指导尽快完成归档，重点关注超期严重项目

📁 归档管理要求

归档时限标准：

  • 采购归档：结果公示后40天内完成归档
  • 合同归档：合同签订后30天内完成归档
  • 结算归档：结算完成后及时归档

超期分级管理：

  • 轻度超期：超期1-15天，常规提醒
  • 中度超期：超期16-30天，重点跟踪
  • 严重超期：超期30天以上，专项处理

4.3 指导录入数据齐全性关注

✅ 数据齐全性标准

关键字段清单：

  • 项目模块：项目编码、项目名称、项目负责人、项目状态
  • 采购模块：采购编号、项目名称、采购方式、预算金额、中标金额
  • 合同模块：合同编号、合同名称、甲方乙方、合同金额、签订日期
  • 付款模块：付款编号、合同编号、付款金额、付款日期
  • 评价模块：评价编号、合同编号、供应商名称、评分

4.4 指导关注工作周期，提高工作效率

⏱️ 工作周期分析

周期定义：

  • 采购周期：需求审批日期 → 中标通知发放日期
  • 合同周期：中标结果公示日期 → 合同签订日期
  • 结算周期：合同签订日期 → 结算完成日期

周期标准：

  • 直接采购：15天
  • 单一来源：25天
  • 询价/竞价/比选：35天
  • 竞争性谈判：35天
  • 公开招标/邀请招标：45天

4.5 数据排行与绩效评估

🏆 项目数据排行

排行维度：

  • 采购规模排行：按采购金额排序
  • 合同数量排行：按合同数量排序
  • 付款进度排行：按付款完成率排序
  • 归档及时率排行：按归档及时率排序
  • 综合绩效排行：多维度综合评分排行

4.6 付款与结算管理现状

📋 当前管理状态

已纳入管理：

  • 付款更新管理：付款记录的更新和状态跟踪
  • 付款数据统计：付款金额、进度、趋势分析

暂未纳入管理：

  • 付款归档管理：付款资料的归档状态跟踪
  • 结算归档管理：结算资料的归档状态跟踪
  • 付款流程监控：付款审批流程的全程监控

5. 应用范围与推广计划

5.1 应用范围界定

✅ 当前应用范围

部门内部：

  • 合约管理部：全面使用系统所有功能
  • 财务部门：重点使用付款和结算功能
  • 项目管理部：重点使用项目和合同管理功能

其他项目数据更新：

  • 各项目部：有数据更新时在系统中更新
  • 新项目数据：新产生的业务数据及时录入系统
  • 历史数据整理：逐步整理重要历史数据

🔄 推广策略调整

阶段性推广：

  • 第一阶段：部门内部稳定使用，验证系统功能和数据准确性
  • 第二阶段：系统稳定且数据无误后，逐步推广到项目部
  • 第三阶段：全面推广应用，实现全流程数字化管理

数据验证机制：

  • 导出检查：各部门可导出数据进行检查验证
  • 汇总核对：通过数据汇总功能检查数据完整性
  • 交叉验证：与现有Excel数据进行交叉验证
  • 问题反馈：发现问题及时反馈并修正

5.2 数据验证与质量保证

🔍 数据验证方法

导出数据检查：

  • 各部门可定期导出系统数据进行检查
  • 对比导出数据与原始记录的一致性
  • 验证数据汇总计算的准确性
  • 检查数据格式和完整性

5.3 数据备份要求

💾 备份责任分工

部门层面备份：

  • 责任人：谷工、张工
  • 备份频率：每周五执行
  • 备份内容：采购和成本相关数据
  • 备份位置：共享盘固定位置

数据更新备份：

  • 各项目部：有数据更新时及时备份
  • 备份时机：重要数据录入后
  • 备份内容：更新的业务数据
  • 备份验证：备份后验证文件完整性

5.4 推广实施计划

📅 稳步推进策略

当前阶段：部门内部稳定使用

  • 重点确保部门内部系统使用稳定
  • 验证系统各项功能的可靠性
  • 检查数据准确性和完整性
  • 收集用户反馈并优化系统

后续阶段：条件成熟后推广

  • 系统运行稳定且数据无误后
  • 逐步推广到各项目部使用
  • 提供系统操作培训和技术支持
  • 建立项目数据录入和更新机制
