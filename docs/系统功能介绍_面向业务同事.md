# 项目采购与成本管理系统 – 功能介绍（面向业务同事）

> 本文面向非技术同事，侧重说明系统能帮你做什么、有哪些模块、典型使用场景是什么。

## 1. 系统定位

项目采购与成本管理系统是一套 **把 Excel 里的分散信息集中到一个系统里管理** 的工具，覆盖：

- 从立项开始的 **项目维度管理**；
- 全流程的 **采购、合同、付款、结算** 管理；
- 针对供应商的 **履约评价与管理**；
- 各类统计报表和 **监控驾驶舱**（归档、周期、数据完整性等）。

可以理解为：

- 把原本散落在多份 Excel 里的数据集中到一个“账本”；
- 自动计算常见指标（金额、节约率、结算率等）；
- 用图表和统计表格帮助你做决策、盘点风险。

## 2. 核心业务模块

### 2.1 项目管理

**目的：** 给每个采购/合同归一个“项目家”，方便按项目维度汇总统计。

主要信息包括：

- 项目编码、项目名称；
- 项目状态（在建、已完结等）；
- 项目负责人及联系方式；
- 项目下挂的采购、合同、付款、结算总览。

**典型使用：**

- 在“项目列表”中按年份或项目负责人筛选，查看某个年度所有在建项目；
- 打开“项目详情”后，可以看到该项目下的采购数量、合同总额、累计付款等汇总数据。

### 2.2 采购管理

**目的：** 管理从需求提出到中标结果的全过程。

主要信息：

- 招采编号、采购项目名称；
- 采购单位、采购方式、采购类别；
- 预算金额、控制价、中标金额；
- 关键时间节点：公告发布时间、候选人公示结束时间、结果公示发布时间、中标通知书发放日期等；
- 需求部门、采购经办人、平台信息等。

**典型使用：**

- 录入或导入每一条采购信息，方便后续关联合同和统计；
- 在统计页面查看按采购方式、类别维度的金额占比、节约率等。

### 2.3 合同管理

**目的：** 管理主合同及补充协议，追踪合同金额和履约状态。

主要信息：

- 合同编号（主键）、合同名称；
- 合同序号（支持类似 `2025-PGCHT-013-001` 序列表达）；
- 文件定位（主合同/补充协议）、合同来源（采购合同/直接签订等）；
- 关联项目、关联采购、关联主合同；
- 合同金额、签订日期、合同类别；
- 甲方/乙方及联系人信息；
- 资料归档日期等。

**典型使用：**

- 从项目维度进入，查看该项目下所有合同及补充协议，了解总合同额与付款进度；
- 在监控驾驶舱中查看合同归档情况、超期风险等。

### 2.4 付款管理

**目的：** 记录与合同相关的每一笔付款，支撑付款进度、结算情况统计。

主要信息：

- 付款编号、付款金额、付款日期；
- 关联合同及其金额；
- 是否已结算、结算金额。

**典型使用：**

- 按项目、合同、时间区间筛选付款记录；
- 在统计页面查看某项目/某供应商的累计付款金额和付款次数。

### 2.5 结算管理

**目的：** 管理与主合同对应的结算信息，关注结算金额与合同金额差异。

主要信息：

- 关联合同、最终结算金额；
- 是否已结算；
- 差异金额和差异率（在统计模块体现）。

**典型使用：**

- 在“结算统计详情”中了解合同额 vs 结算额的差异；
- 快速筛出长期未结算的合同，辅助风险提示。

### 2.6 供应商履约评价

**目的：** 对供应商在不同时间点的履约表现进行打分，形成综合评价。

主要内容：

- 每个合同对应一个或多个评价记录；
- 年度评价得分（如 2023 年度、2024 年度）；
- 不定期评价（第 1 次、第 2 次……）；
- 综合评分、末次评价得分。

**典型使用：**

- 在供应商管理表中查看某供应商历史履约记录；
- 为供应商评优或黑名单管理提供数据支撑。

## 3. 监控与统计功能

### 3.1 监控驾驶舱

**目标：** 用图表和指标给管理层一个“抬头就能看懂”的总览。

主要视图包括：

- 归档监控：关注招采结果到归档的时间差，统计超期情况；
- 周期监控：采购周期、合同签订周期等；
- 更新监控：数据最近更新时间，是否存在长时间未更新的项目/模块；
- 完整性检查：哪些项目/合同信息不完整（缺关键字段）。

### 3.2 统计报表

系统提供多个维度的统计报表：

- 采购统计：按采购方式、类别、年度等维度统计数量与金额、节约额和节约率；
- 合同统计：合同数量、合同总额、支付次数和支付比例；
- 付款统计：按项目、供应商、时间维度统计付款情况；
- 结算统计：统计合同 vs 结算的差异额与差异率；
- 排名统计：如采购按时归档率排名、项目执行情况排名等。

所有统计都支持：

- 年份筛选（按年度分析）；
- 项目筛选（关注某几个重点项目）；
- 导出为 Excel（详情表），便于进一步加工或汇报。

## 4. 数据导入与导出

### 4.1 Excel 导出

支持两类导出：

- 单项目导出：
  - 为每个项目生成一个包含“采购表、合同表、付款表、结算表、供应商管理表”的 Excel 文件；
  - 文件名自动带项目名称和时间戳，便于归档；
- 多项目导出：
  - 自动为每个项目生成一个 Excel，并打包成 ZIP；
  - 当项目数量很多时，会采用后台异步处理方式，生成完成后通过邮件通知。

### 4.2 Excel 导入

- 支持从标准模板导入采购/合同/付款数据；
- 导入时会进行字段校验和业务校验（比如是否存在对应项目）；
- 出错时会给出汇总的错误信息，便于逐项修正再导入。

> 模板和字段规则在《数据模型使用手册》中有详细说明，业务同事可以参考模板填写。

## 5. PDF 智能导入

系统提供 PDF 智能导入模块，用于从 PDF 文档中识别项目、合同相关信息：

- 自动识别是否为合法 PDF（文件类型 + 结构完整性检查）；
- 支持从 PDF 中抽取关键信息，用于辅助录入；
- 对于识别不确定的内容，会要求人工确认。

这部分功能主要是为了减少录入工作量，但不会替代人工终审。

## 6. 权限与安全

### 6.1 登录与权限

- 所有业务功能都需要登录之后才能访问；
- 系统支持“角色 + 权限”的方式进行访问控制：
  - 例如：只能某些角色可以进行数据导出、批量删除等敏感操作；
- 操作行为会记录到审计日志中（包括谁、在什么时候，对什么数据做了什么操作）。

### 6.2 安全与合规

- 针对 Web 安全风险（CSRF、会话安全、Cookie 安全等）做了统一配置；
- 默认使用 HTTPS 协议，保证局域网传输安全；
- 对上传文件（尤其是 PDF）进行严格校验，避免恶意文件注入。

## 7. 日常使用建议

- 尽量在系统中完成数据录入和更新，避免“系统 + Excel 多头维护”；
- 统一使用系统定义的编码（项目编码、采购编号、合同编号等），防止混乱；
- 如需新增统计口径或报表，可以先和系统维护人沟通，再统一配置，避免各自单独维护。
