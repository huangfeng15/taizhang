## 1. 系统概述与价值定位

### 1.1 系统是什么？

项目采购与成本管理系统是一套**将分散的Excel数据集中化、流程化管理的数字化平台**，覆盖从项目立项到最终结算的全生命周期管理。

**核心价值：**
- ✅ **数据集中管理**：告别多份Excel分散维护的混乱局面
- ✅ **流程标准化**：建立规范的采购、合同、付款管理流程
- ✅ **实时监控**：随时掌握项目进展和风险状况
- ✅ **决策支持**：通过数据分析支撑管理决策

### 1.2 系统解决的核心痛点

| 传统Excel管理方式 | 系统化管理方式 |
|-------------------|----------------|
| 🔴 数据分散在多个Excel文件，查找困难 | ✅ 统一数据库存储，快速检索 |
| 🔴 重复录入，容易出错 | ✅ 一次录入，多处复用 |
| 🔴 统计分析耗时，容易遗漏 | ✅ 自动统计，实时生成报表 |
| 🔴 缺乏流程管控，责任不清 | ✅ 流程节点清晰，责任明确 |
| 🔴 历史数据追溯困难 | ✅ 完整操作日志，全程可追溯 |

### 1.3 系统带来的具体价值

#### 1.3.1 效率提升价值
- **数据录入效率提升60%**：通过智能导入和关联关系自动填充
- **统计报表生成时间从小时级缩短到分钟级**
- **减少重复性数据核对工作**

#### 1.3.2 管理规范价值
- **建立统一的数据标准和编码规范**
- **实现业务流程的标准化和规范化**
- **提供完整的审计追踪机制**

#### 1.3.3 决策支持价值
- **实时掌握项目整体进展状况**
- **及时发现潜在风险和问题**
- **基于数据的科学决策支持**

## 2. 核心业务模块详解

### 2.1 项目管理模块

#### 2.1.1 业务定位
项目管理是整个系统的**顶层组织单元**，为所有采购、合同、付款等业务数据提供项目维度的归类和管理。

#### 2.1.2 核心功能
- **项目基础信息管理**：项目编码、名称、负责人、状态等
- **项目关联数据总览**：实时显示项目下的采购、合同、付款汇总
- **项目进展监控**：跟踪项目执行状态和关键节点

#### 2.1.3 业务价值
- 📊 **项目全景视图**：一个页面了解项目整体情况
- 🎯 **责任明确**：清晰的项目负责人和联系方式
- 📈 **进展跟踪**：实时掌握项目执行状态

#### 2.1.4 关键字段说明
| 字段名称 | 业务含义 | 填写要求 |
|---------|---------|---------|
| 项目编码 | 项目唯一标识 | 格式：PRJ + 年份 + 3位序号，如：PRJ2025001 |
| 项目名称 | 项目正式名称 | 必填，使用官方全称 |
| 项目负责人 | 项目主要责任人 | 必填，填写具体姓名 |
| 项目状态 | 当前执行状态 | 选择：进行中/已完成/已暂停/已取消 |

### 2.2 采购管理模块

#### 2.2.1 业务定位
采购管理模块记录从**采购需求提出到中标结果确定**的完整过程，是合同管理的前置环节。

#### 2.2.2 核心功能
- **采购全流程记录**：从需求审批到中标通知的完整时间线
- **采购方式管理**：支持9种标准采购方式
- **预算控制分析**：预算金额、控制价、中标价的对比分析
- **采购周期监控**：跟踪采购各环节的时间效率

#### 2.2.3 业务价值
- 💰 **成本控制**：通过预算对比实现采购成本有效控制
- ⏱️ **效率监控**：采购周期分析帮助优化采购流程
- 📋 **合规管理**：完整的采购过程记录确保合规性

#### 2.2.4 采购方式详解
系统支持9种标准采购方式，分为法定方式和扩展方式：

**法定方式（6种）：**
1. **公开招标**：适用于大额政府采购项目
2. **邀请招标**：适用于技术复杂或特殊要求项目
3. **竞争性谈判**：适用于招标失败或技术复杂项目
4. **竞争性磋商**：适用于政府购买服务项目
5. **单一来源采购**：适用于唯一供应商或紧急情况
6. **询价采购**：适用于规格统一、货源充足的货物

**扩展方式（3种）：**
7. **直接采购**：适用于小额零星采购
8. **竞价采购**：适用于标准化商品采购
9. **比选**：适用于国际组织贷款项目

#### 2.2.5 关键业务指标
- **节约率** = (预算金额 - 中标金额) / 预算金额 × 100%
- **采购周期** = 中标通知发放日期 - 需求审批完成日期
- **及时完成率**：按时完成的采购数量占比

### 2.3 合同管理模块

#### 2.3.1 业务定位
合同管理是系统的**核心业务模块**，管理主合同及补充协议，是付款和结算的基础。

#### 2.3.2 核心功能
- **合同全生命周期管理**：从签订到归档的完整过程
- **补充协议管理**：支持主合同-补充协议的关联关系
- **合同金额管控**：合同金额、付款比例的实时计算
- **合同归档监控**：跟踪合同资料的归档状态

#### 2.3.3 业务价值
- 📝 **风险管控**：通过合同金额和付款比例监控财务风险
- 🔗 **关系管理**：清晰的合同关联关系避免管理混乱
- 📁 **档案管理**：完整的合同资料归档确保合规性

#### 2.3.4 合同类型说明
系统支持三种合同类型：

**主合同**：
- 与采购项目直接关联的原始合同
- 是补充协议和结算的基础
- 一个采购项目可以产生多个主合同

**补充协议**：
- 对主合同的变更和补充
- 必须关联到具体的主合同
- 金额会累加到主合同总额中

**解除协议**：
- 用于解除原合同关系
- 必须关联到具体的主合同
- 通常涉及金额调整

#### 2.3.5 关键业务指标
- **付款比例** = 累计付款金额 / (合同金额 + 补充协议金额) × 100%
- **归档及时率**：按时归档的合同数量占比
- **合同签订周期** = 合同签订日期 - 中标结果公示日期

### 2.4 付款管理模块

#### 2.4.1 业务定位
付款管理模块记录**每一笔具体的付款交易**，是资金执行的具体体现。

#### 2.4.2 核心功能
- **付款记录管理**：详细记录每笔付款的金额、时间、用途
- **付款进度跟踪**：实时计算付款比例和剩余金额
- **结算标记**：标记已办理结算的付款
- **付款编号自动生成**：按规则自动生成唯一付款编号

#### 2.4.3 业务价值
- 💵 **资金管控**：实时掌握资金支付进度和剩余预算
- 📊 **执行分析**：付款进度分析支持资金安排决策
- 🔍 **追溯管理**：完整的付款记录支持审计追溯

#### 2.4.4 付款编号规则
付款编号采用**合同标识-FK-序号**格式：
- **合同标识**：使用合同序号或合同编号
- **FK**：固定标识符，表示付款
- **序号**：3位数字，按付款日期排序生成

**示例**：HT2025001-FK-001 表示合同HT2025001的第1笔付款

#### 2.4.5 关键业务指标
- **付款完成率** = 累计付款金额 / 应付总金额 × 100%
- **平均付款周期**：从合同签订到付款完成的平均时间
- **结算率** = 已结算付款金额 / 总付款金额 × 100%

### 2.5 结算管理模块

#### 2.5.1 业务定位
结算管理模块记录**主合同的最终结算**，是项目成本控制的最终环节。

#### 2.5.2 核心功能
- **最终结算记录**：记录主合同的最终结算金额
- **差异分析**：自动计算合同额与结算额的差异
- **结算完成跟踪**：监控结算进度和完成情况

#### 2.5.3 业务价值
- 🎯 **成本控制**：最终结算金额反映实际成本控制效果
- 📊 **差异分析**：合同额与结算额差异分析支持成本优化
- ✅ **项目闭环**：结算完成标志项目成本管理的闭环

#### 2.5.4 关键业务指标
- **结算差异率** = (结算金额 - 合同金额) / 合同金额 × 100%
- **结算完成率** = 已结算合同数量 / 应结算合同数量 × 100%
- **平均结算周期**：从合同签订到结算完成的平均时间

### 2.6 供应商履约评价模块

#### 2.6.1 业务定位
供应商履约评价模块记录**供应商在项目中的表现**，为供应商管理提供数据支撑。

#### 2.6.2 核心功能
- **履约评价记录**：记录供应商的具体评价信息
- **评分管理**：支持多维度评分和综合评分
- **历史记录查询**：查看供应商的历史履约表现
- **评价趋势分析**：分析供应商表现的变化趋势

#### 2.6.3 业务价值
- 🏆 **供应商管理**：基于数据的供应商选择和管理
- 📈 **质量监控**：通过评价监控供应商服务质量
- 🔄 **持续改进**：评价结果推动供应商持续改进

#### 2.6.4 评价类型说明
- **末次评价**：项目完成后的最终评价
- **履约过程评价**：项目执行过程中的阶段性评价

#### 2.6.5 评分等级标准
- **优秀**：综合评分 ≥ 90分
- **良好**：80分 ≤ 综合评分 < 90分
- **合格**：70分 ≤ 综合评分 < 80分
- **不合格**：综合评分 < 70分

## 3. 系统操作手册

### 3.1 系统登录与界面概览

#### 3.1.1 系统访问
1. **打开浏览器**：推荐使用Chrome、Edge等现代浏览器
2. **输入系统地址**：`https://系统IP地址:3500`
3. **处理安全警告**：点击"继续访问"或"信任此站点"
4. **输入登录信息**：用户名和密码由系统管理员分配

#### 3.1.2 主界面布局
系统主界面采用**顶部导航 + 左侧菜单**的布局：

**顶部导航栏**：
- 全局年份筛选器：快速切换查看不同年度数据
- 全局项目筛选器：按项目筛选查看相关数据
- 用户信息：显示当前登录用户和退出按钮

**左侧菜单栏**：
- 🏠 **仪表盘**：系统概览和关键指标
- 📁 **项目管理**：项目列表和详情查看
- 🛒 **采购管理**：采购记录的增删改查
- 📄 **合同管理**：合同信息和补充协议管理
- 💰 **付款管理**：付款记录和结算信息
- ⭐ **供应商评价**：供应商履约评价管理
- 📊 **统计分析**：各类统计报表和图表
- 🔍 **监控中心**：归档、周期、完整性监控
- 📤 **数据导入导出**：Excel批量导入导出功能

### 3.2 基础操作指南

#### 3.2.1 数据查看操作

**列表页面操作**：
1. **筛选功能**：页面顶部的筛选器支持多条件组合筛选
2. **排序功能**：点击列标题可进行升序/降序排序
3. **搜索功能**：右上角搜索框支持关键词快速搜索
4. **分页功能**：底部分页器支持页面跳转和每页条数设置

**详情页面操作**：
1. **基本信息查看**：顶部显示记录的基本信息
2. **关联数据查看**：中部显示相关的采购、合同、付款等数据
3. **操作历史查看**：底部显示该记录的操作历史

#### 3.2.2 数据录入操作

**新增记录**：
1. 点击页面右上角的"新增"按钮
2. 填写必填字段（标有红色星号*）
3. 选择或填写可选字段
4. 点击"保存"按钮完成录入

**编辑记录**：
1. 在列表页面找到要编辑的记录
2. 点击操作列的"编辑"按钮
3. 修改需要更新的字段
4. 点击"保存"按钮完成修改

**删除记录**：
1. 在列表页面找到要删除的记录
2. 点击操作列的"删除"按钮
3. 在确认对话框中点击"确认删除"

#### 3.2.3 高级功能操作

**批量操作**：
1. 在列表页面勾选需要批量操作的记录
2. 点击上方的批量操作按钮
3. 按提示完成批量操作

**数据导出**：
1. 在列表页面设置好筛选条件
2. 点击"导出Excel"按钮
3. 选择导出范围（当前页/全部）
4. 下载生成的Excel文件

**数据导入**：
1. 点击"导入Excel"按钮
2. 下载标准模板文件
3. 按模板格式填写数据
4. 上传填写好的Excel文件
5. 查看导入结果和错误信息

### 3.3 各模块详细操作

#### 3.3.1 项目管理操作

**新增项目**：
1. 进入"项目管理"页面
2. 点击"新增项目"按钮
3. 填写项目信息：
   - 项目编码：系统自动生成，格式为PRJ+年份+3位序号
   - 项目名称：填写项目官方全称
   - 项目负责人：选择或输入负责人姓名
   - 项目状态：选择当前状态
   - 项目描述：填写项目详细说明
4. 点击"保存"完成创建

**查看项目详情**：
1. 在项目列表中点击项目名称
2. 查看项目基本信息
3. 查看关联的采购、合同、付款汇总
4. 查看项目进展统计图表

#### 3.3.2 采购管理操作

**新增采购记录**：
1. 进入"采购管理"页面
2. 点击"新增采购"按钮
3. 填写采购信息：
   - 采购编码：系统自动生成
   - 关联项目：选择所属项目（可选）
   - 采购名称：填写采购项目全称
   - 采购单位：选择发起采购的部门
   - 采购方式：从9种标准方式中选择
   - 预算金额：填写项目预算
   - 中标金额：填写实际中标金额
   - 关键日期：填写各环节日期
4. 点击"保存"完成创建

**采购周期分析**：
1. 在采购列表中点击"周期分析"
2. 查看采购各环节的时间分析
3. 查看与标准周期的对比
4. 识别超期环节和原因

#### 3.3.3 合同管理操作

**新增主合同**：
1. 进入"合同管理"页面
2. 点击"新增合同"按钮
3. 填写合同信息：
   - 合同编码：系统自动生成
   - 关联项目：选择所属项目
   - 关联采购：选择对应的采购项目
   - 文件定位：选择"主合同"
   - 合同名称：填写合同正式名称
   - 合同金额：填写合同总金额
   - 签订日期：选择合同签订日期
   - 甲乙双方信息：填写详细的联系信息
4. 点击"保存"完成创建

**新增补充协议**：
1. 在主合同详情页面点击"新增补充协议"
2. 系统自动继承主合同的关联关系
3. 填写补充协议的特定信息：
   - 协议名称：描述补充协议内容
   - 协议金额：填写补充协议金额
   - 签订日期：选择协议签订日期
4. 点击"保存"完成创建

**付款进度查看**：
1. 在合同详情页面查看付款进度条
2. 查看累计付款金额和付款比例
3. 查看详细的付款记录列表
4. 识别未付款项和逾期风险

#### 3.3.4 付款管理操作

**新增付款记录**：
1. 进入"付款管理"页面
2. 点击"新增付款"按钮
3. 填写付款信息：
   - 关联合同：选择对应的合同
   - 付款金额：填写实际付款金额
   - 付款日期：选择付款日期
   - 结算标记：勾选如果已办理结算
   - 结算金额：填写结算价（如已结算）
4. 点击"保存"完成创建

**付款编号说明**：
- 付款编号自动生成，格式为：合同标识-FK-序号
- 序号按付款日期排序，确保编号的有序性
- 如需修改付款日期，编号会自动重新排序

#### 3.3.5 供应商评价操作

**新增评价记录**：
1. 进入"供应商评价"页面
2. 点击"新增评价"按钮

## 4. 数据管理最佳实践

### 4.1 数据录入规范

#### 4.1.1 编码规范
系统采用统一的编码规范，确保数据的唯一性和规范性：

**项目编码**：
- 格式：PRJ + 年份 + 3位序号
- 示例：PRJ2025001、PRJ2025002
- 注意：年份使用4位数字，序号从001开始

**采购编码**：
- 格式：GC + 年份 + 3位序号
- 示例：GC2025001、GC2025002
- 注意：GC代表"采购"的拼音缩写

**合同编码**：
- 格式：HT + 年份 + 3位序号
- 示例：HT2025001、HT2025002
- 注意：HT代表"合同"的拼音缩写

**付款编码**：
- 格式：合同标识 + FK + 3位序号
- 示例：HT2025001-FK-001
- 注意：FK代表"付款"，序号按付款日期排序

#### 4.1.2 日期格式规范
- **标准格式**：YYYY-MM-DD
- **示例**：2025-01-15、2025-12-31
- **注意事项**：
  - 所有日期字段必须使用标准格式
  - 避免使用中文日期格式
  - 确保日期的逻辑合理性

#### 4.1.3 金额格式规范
- **标准格式**：数字格式，保留2位小数
- **示例**：123456.78、1000000.00
- **注意事项**：
  - 不使用千分位分隔符
  - 金额必须大于等于0
  - 预算金额≥中标金额≥结算金额（正常情况下）

#### 4.1.4 文本字段规范
- **项目名称**：使用官方全称，避免简称
- **供应商名称**：使用营业执照上的全称
- **人员姓名**：使用真实姓名，避免昵称
- **联系方式**：格式统一，包含区号
