## 1. 系统开发背景与价值

### 1.1 为什么要开发这个系统？

在系统开发之前，我们的采购与成本管理面临以下**核心痛点**：

#### 🔴 传统Excel管理方式的困境

| 痛点问题 | 具体表现 | 影响 |
|---------|---------|------|
| **数据分散混乱** | 多个Excel文件，版本不统一 | 查找困难，数据不一致 |
| **重复录入工作量大** | 相同信息在多个表格重复填写 | 效率低下，容易出错 |
| **缺乏关联关系** | 项目、采购、合同、付款数据孤立 | 无法形成完整业务链条 |
| **统计分析困难** | 手工汇总统计，耗时耗力 | 决策滞后，容易遗漏 |
| **数据质量难保证** | 格式不统一，校验困难 | 数据错误影响决策 |
| **历史追溯困难** | 缺乏完整的操作记录 | 问题难以分析和改进 |

#### ✅ 系统化管理的价值

**1. 提高工作效率**
- **减少重复录入**：通过外键关联，一次录入多处复用
- **统计报表自动化**：自动生成各类统计报表，减少手工统计工作
- **减少重复性工作**：自动计算、自动关联、自动校验

**2. 减少错误和不统一**
- **统一数据标准**：编码规范、格式统一、字段标准
- **自动数据校验**：必填字段检查、逻辑关系验证、格式规范检查
- **版本控制**：单一数据源，避免多版本混乱

**3. 减少重复信息录入**
- **外键关联机制**：项目信息自动关联到采购、合同
- **数据继承**：补充协议自动继承主合同信息
- **智能填充**：基于关联关系自动填充相关信息

**4. 统一管理平台**
- **集中化数据存储**：所有数据统一管理，避免分散
- **标准化业务流程**：建立规范的业务管理流程
- **权限分级管理**：不同角色不同权限，确保数据安全

### 1.2 挖掘数据价值，让台账发挥真正价值

传统Excel台账只是**数据的记录工具**，而我们的系统要让台账成为**业务指导工具**：

#### 📊 从"记录"到"洞察"
- **实时监控**：随时掌握项目进展和风险状况
- **趋势分析**：通过历史数据分析发展趋势
- **异常预警**：自动识别异常情况并及时提醒
- **绩效评估**：基于数据的客观绩效评价

#### 🎯 从"被动"到"主动"
- **主动管理**：基于数据预警主动采取措施
- **精准决策**：基于数据分析做出科学决策
- **持续改进**：通过数据反馈持续优化流程
- **价值创造**：让数据成为管理改进的驱动力

## 2. 系统运行环境与安全

### 2.1 系统运行方式

#### 🖥️ 部署架构
- **运行环境**：系统部署在本地电脑，每天开机自动运行
- **访问方式**：仅支持局域网内访问，确保数据安全
- **服务地址**：`https://系统IP地址:3500`
- **访问协议**：HTTPS加密传输，保障数据传输安全

#### 📱 访问要求
- **网络要求**：必须连接公司局域网
- **浏览器要求**：推荐使用Chrome、Edge等现代浏览器
- **系统兼容**：支持Windows、Mac、Linux等操作系统

### 2.2 安全管理机制

#### 🔐 登录安全
- **登录有效期**：12小时自动过期，需要重新登录
- **会话管理**：超时自动退出，防止未授权访问
- **密码保护**：定期更换密码，确保账户安全

#### 🛡️ 数据安全
- **权限控制**：暂未分配细粒度权限，所有用户权限相同
- **操作限制**：**禁止进行删除数据库操作**，确保数据安全
- **审计日志**：记录所有操作行为，支持问题追溯

#### 💾 数据备份
- **自动备份**：系统每日自动进行数据备份
- **手动备份**：支持手动导出数据备份
- **备份位置**：备份文件存储到共享盘指定位置
- **备份策略**：保留最近30天的备份文件

### 2.3 使用注意事项

#### ⚠️ 重要提醒
1. **仅限局域网访问**：外网无法访问系统
2. **12小时登录有效期**：超时需要重新登录
3. **禁止删除操作**：没有经过沟通切勿尝试删除数据库数据，因为现有数据可能已做了一些微小调整。
4. **及时备份数据**：重要操作前建议手动备份

#### 🔧 故障处理
- **无法访问**：检查网络连接，确认系统是否运行
- **登录问题**：联系系统管理员重置密码
- **数据异常**：立即联系技术支持处理
- **系统故障**：等待管理员修复，期间使用Excel临时记录

## 3. 系统模块功能介绍

### 3.1 核心业务模块

#### 🏗️ 项目管理模块
**功能定位**：项目的顶层组织单元，为所有业务数据提供项目维度管理

**主要功能**：
- 项目基础信息管理（编码、名称、负责人、状态）
- 项目关联数据总览（采购、合同、付款汇总）
- 项目进展状态跟踪

**导入模板**：
- **模板名称**：项目信息导入模板
- **文件格式**：Excel (.xlsx)
- **关键字段**：项目编码、项目名称、项目负责人、项目状态
- **导入规则**：项目编码唯一，项目名称必填

#### 🛒 采购管理模块
**功能定位**：记录从需求提出到中标结果的完整采购过程

**主要功能**：
- 采购全流程记录（需求→公告→开标→中标→归档）
- 采购方式管理（支持9种标准采购方式）
- 预算控制分析（预算、控制价、中标价对比）
- 采购周期监控

**导入模板**：
- **模板名称**：采购信息导入模板_{year}.xlsx
- **文件格式**：Excel (.xlsx)
- **关键字段**：采购编号、项目名称、采购方式、预算金额、中标金额、各环节日期
- **导入规则**：
  - 采购编号唯一
  - 日期格式：YYYY-MM-DD
  - 金额单位：元
  - 采购方式必须从标准选项中选择

#### 📄 合同管理模块
**功能定位**：管理主合同及补充协议，是付款和结算的基础

**主要功能**：
- 合同全生命周期管理
- 补充协议关联管理
- 合同金额和付款比例计算
- 合同归档状态跟踪

**导入模板**：
- **模板名称**：合同信息导入模板_{year}.xlsx
- **文件格式**：Excel (.xlsx)
- **关键字段**：合同编号、合同名称、项目名称、甲方乙方、合同金额、签订日期
- **导入规则**：
  - 合同编号唯一
  - 补充协议必须关联主合同
  - 采购合同必须关联采购项目
  - 文件定位：主合同/补充协议/解除协议

#### 💰 付款管理模块
**功能定位**：记录每一笔付款交易，跟踪资金执行情况

**主要功能**：
- 付款记录管理
- 付款进度跟踪
- 结算标记管理
- 付款编号自动生成

**导入模板**：
- **模板名称**：付款信息导入模板_{year}.xlsx
- **文件格式**：Excel (.xlsx)
- **关键字段**：付款编号、合同编号、付款金额、付款日期、结算标记
- **导入规则**：
  - 付款编号唯一
  - 必须关联有效合同
  - 付款金额必须大于0
  - 付款日期格式：YYYY-MM-DD

#### ⭐ 供应商履约评价模块
**功能定位**：记录供应商履约表现，为供应商管理提供数据支撑

**主要功能**：
- 履约评价记录
- 多维度评分管理
- 历史记录查询
- 评价趋势分析

**导入模板**：
- **模板名称**：供应商履约评价导入模板_{year}.csv
- **文件格式**：CSV (13列格式)
- **关键字段**：序号、合同序号、履约综合评价得分、各年度评价得分、备注
- **导入规则**：
  - 合同序号必须在系统中存在
  - 评分范围：0-100分
  - 支持综合评分自动计算（末次60%+过程40%）

### 3.2 数据导入导出功能

#### 📥 数据导入
**导入方式**：
1. **页面导入**：在各模块页面点击"导入Excel"按钮

#### 📤 数据导出
**导出方式**：
1. **页面导出**：在各列表页面点击"导出Excel"按钮
2. **筛选导出**：支持按筛选条件导出特定数据
3. **全量导出**：支持导出全部数据（大数据量时异步处理）
4. **备份导出**：定期导出数据进行备份

**导出格式**：
- **Excel格式**：.xlsx格式，便于后续处理
- **包含字段**：完整的业务字段和关联信息
- **格式规范**：标准化的日期、金额、编码格式

## 4. 系统指导工作应用

### 4.1 指导及时更新台账数据

#### 📅 数据更新及时性要求
**更新频率要求**：
- **日常数据**：业务完成后24小时内更新
- **重要节点**：关键节点完成后立即更新
- **定期检查**：每周检查数据更新情况

**关键更新节点**：
- **采购模块**：需求审批、公告发布、开标、中标、归档
- **合同模块**：合同签订、补充协议、付款节点、归档
- **付款模块**：每笔付款完成后立即更新
- **评价模块**：评价完成后及时录入

#### ⏰ 及时性监控机制
**系统监控功能**：
- **更新监控**：显示各模块最近更新时间
- **超期预警**：识别长时间未更新的数据
- **更新统计**：统计更新频率和及时率
- **责任追踪**：明确各数据的更新责任人

**管理措施**：
- **定期检查**：每周检查数据更新情况
- **超期提醒**：对超期未更新数据进行提醒
- **绩效考核**：将数据及时性纳入绩效考核
- **持续改进**：分析更新延迟原因并改进

### 4.2 指导尽快完成归档，重点关注超期严重项目

#### 📁 归档管理要求
**归档时限标准**：
- **采购归档**：结果公示后40天内完成归档
- **合同归档**：合同签订后30天内完成归档
- **结算归档**：结算完成后及时归档

**归档内容要求**：
- **完整性**：所有相关资料齐全
- **规范性**：按照标准格式整理
- **准确性**：信息准确无误
- **及时性**：在规定时限内完成

#### 🚨 超期项目重点关注
**超期分级管理**：
- **轻度超期**：超期1-15天，常规提醒
- **中度超期**：超期16-30天，重点跟踪
- **严重超期**：超期30天以上，专项处理

**重点关注措施**：
- **每日监控**：严重超期项目每日跟踪
- **专项会议**：定期召开超期项目专题会议
- **责任到人**：明确超期项目处理责任人
- **资源倾斜**：优先配置资源处理超期项目

**系统支持功能**：
- **归档监控**：实时显示归档状态和超期情况
- **超期预警**：自动识别和预警超期项目
- **归档排名**：按归档及时率进行排名
- **处理跟踪**：记录超期项目处理进展

### 4.4 指导关注工作周期，提高工作效率

#### ⏱️ 工作周期分析
**周期定义**：
- **采购周期**：需求审批日期 → 中标通知发放日期
- **合同周期**：中标结果公示日期 → 合同签订日期
- **结算周期**：合同签订日期 → 结算完成日期

**周期标准**：

- **询价/竞价/比选**：35天

#### 📈 周期效率监控

- **平均周期**：各采购方式的平均执行周期
- **周期分布**：周期分布情况（<30天、30-60天、60-90天、>90天）
- **按时完成率**：在标准周期内完成的比例
- **周期趋势**：周期变化趋势分析

**效率提升措施**：
- **瓶颈识别**：识别周期过长的主要环节
- **流程优化**：优化业务流程，减少不必要的环节
- **标准化**：建立标准化的操作流程和时间要求
- **技能提升**：提升相关人员专业技能和效率

#### 💾 备份责任分工

**项目部层面备份**：
- **责任人**：各项目部指定人员
- **备份频率**：每周执行
- **备份内容**：项目部相关数据
- **备份位置**：共享盘项目部文件夹

#### 📋 备份操作规范
**备份操作步骤**：
1. 登录系统，进入数据导出页面
2. 选择需要备份的数据模块
3. 设置时间范围（建议选择全部时间）
4. 执行导出操作
5. 将导出文件保存到共享盘指定位置
6. 验证备份文件完整性

**备份文件命名规范**：
- 格式：`模块名_备份日期.xlsx`
- 示例：`采购数据_20251117.xlsx`
- 要求：日期格式使用YYYYMMDD

**备份验证要求**：
- 每次备份后验证文件完整性
- 定期测试备份文件可用性
- 发现问题及时重新备份
