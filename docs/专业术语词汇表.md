## 1. 核心业务术语

### 1.1 项目与采购

| 术语 | 英文对照 | 定义 | 使用场景 |
| :--- | :--- | :--- | :--- |
| **项目** | Project | 业务的顶层容器，所有采购、合同都归属于一个项目。 | 数据组织、筛选 |
| **采购** | Procurement | 指从需求提出到确定中标供应商的全过程。 | 业务流程、合同前置 |
| **招采编号** | Procurement Code | 采购活动的唯一标识符，如 `GC2025001`。 | 记录查询、数据关联 |
| **采购方式** | Procurement Method | 执行采购所采用的具体方法，如公开招标、单一来源等。 | 流程分类、统计分析 |
| **工作周期** | Work Cycle | 采购和合同业务处理的时间跨度，用于衡量工作效率。 | 绩效评估、效率分析 |
| **采购周期** | Procurement Cycle | 从需求书审批完成到结果公示完成的时间周期。 | 流程管理、时效监控 |
| **合同周期** | Contract Cycle | 从采购结果公示完成到合同签订完成的时间周期。 | 流程管理、时效监控 |
| **按时率** | On-Time Rate | 在规定期限内完成的业务占总业务的比例。 | 绩效评估、流程优化 |
| **中标金额** | Winning Amount | 采购活动最终确定的成交金额。 | 成本核算、资金估算 |

### 1.2 合同与付款

| 术语 | 英文对照 | 定义 | 使用场景 |
| :--- | :--- | :--- | :--- |
| **合同** | Contract | 甲乙双方签订的法律协议。 | 核心业务对象 |
| **合同编号** | Contract Code | 合同的唯一业务标识，如 `HT2025001`。 | 记录查询、数据关联 |
| **合同序号** | Contract Sequence | 用于排序和识别的辅助编号，格式灵活。 | 快速查找、历史数据兼容 |
| **文件定位** | File Positioning | 合同的性质分类，决定其业务规则。 | 区分主合同、补充协议等 |
| **主合同** | Main Contract | 独立的、原始的合同。 | 关联补充协议、结算的基础 |
| **补充协议** | Supplement | 对主合同条款进行补充或修改的协议。 | 金额累加、条款变更 |
| **付款** | Payment | 根据合同约定支付款项的记录。 | 资金流跟踪、财务对账 |
| **付款比例** | Payment Ratio | `累计付款金额 / 基准金额`，反映合同执行进度。 | 风险预警、进度监控 |
| **结算** | Settlement | 合同完成后的最终金额确认。 | 最终成本核算 |

### 1.3 供应商评价

| 术语 | 英文对照 | 定义 | 使用场景 |
| :--- | :--- | :--- | :--- |
| **供应商履约评价** | Supplier Performance Evaluation | 对供应商在合同执行过程中的表现进行的综合评价。 | 供应商管理、绩效考核 |
| **末次评价** | Final Evaluation | 在合同执行末期对供应商进行的最终评价。 | 合同收尾、供应商评级 |
| **履约过程评价** | Performance Process Evaluation | 在合同执行过程中对供应商进行的阶段性评价。 | 过程监控、持续改进 |
| **综合评分** | Comprehensive Score | 对供应商各方面表现的综合打分，满分100分。 | 评价排序、等级划分 |
| **评价等级** | Evaluation Grade | 根据综合评分划分的供应商表现等级：优秀/良好/合格/不合格。 | 供应商分类管理 |
| **供应商汇总** | Supplier Summary | 供应商所有合同的总体统计数据，包括合同数、金额、评分等。 | 供应商全景视图 |
| **评价趋势** | Evaluation Trend | 供应商历年评价得分的变化趋势，用于分析供应商发展态势。 | 趋势分析、战略决策 |

## 2. 技术与架构术语

### 2.1 系统架构

| 术语 | 英文对照 | 定义 | 代码位置/示例 |
| :--- | :--- | :--- | :--- |
| **混合架构** | Hybrid Architecture | “Django Admin + 自定义前端页面”的架构模式。 | 整个项目 |
| **服务层** | Service Layer | 封装复杂业务逻辑的独立模块，实现视图与逻辑解耦。 | `project/services/` |
| **ORM查询优化** | ORM Optimization | 使用`select_related`, `prefetch_related`, `annotate`等方法减少数据库查询，提升性能。 | `project/views.py` |
| **N+1查询** | N+1 Query Problem | ORM中常见的性能陷阱，在循环中对每个对象都触发一次数据库查询。 | 性能优化的主要目标 |
| **数据库聚合** | DB Aggregation | 使用`Sum`, `Count`等函数，将计算任务下推到数据库执行。 | `annotate()` |

### 2.2 数据处理

| 术语 | 英文对照 | 定义 | 代码位置/示例 |
| :--- | :--- | :--- | :--- |
| **长表** | Long Format | 标准的关系数据表，每行代表一个独立的记录。 | 标准导入格式 |
| **宽表** | Wide Format | 将时间等维度作为列的数据表，常见于Excel。 | 历史数据导入格式 |
| **数据清洗** | Data Cleansing | 在导入前对原始数据进行格式化、修正和标准化的过程。 | `scripts/prepare_import_data.py` |
| **管理命令** | Management Command | 可通过 `python manage.py` 调用的自定义脚本。 | `procurement/management/commands/` |

### 3.2 交互概念

| 术语 | 英文对照 | 定义 | 示例 |
| :--- | :--- | :--- | :--- |
| **级联筛选** | Cascading Select | 一个选择器的值动态改变另一个选择器的可选范围。 | 选择项目后，合同选择器只显示该项目下的合同。 |
| **搜索防抖** | Debouncing | 在用户停止输入一段时间后才触发搜索，防止频繁API调用。 | 智能选择器的搜索框。 |
| **懒加载** | Lazy Loading | 仅在需要时才加载数据，如打开下拉框或滚动到页面底部。 | 智能选择器、分页列表。 |

## 4. 监控与报表术语

| 术语 | 英文对照 | 定义 | 使用场景 |
| :--- | :--- | :--- | :--- |
| **监控驾驶舱** | Monitoring Cockpit | 汇集所有核心监控指标（归档、更新、完整性）的综合视图。 | 宏观状态概览 |
| **工作周期监控** | Work Cycle Monitoring | 对采购周期和合同周期的实时监控，支持按时率统计和预警。 | 绩效评估、流程优化 |
| **归档及时率** | Archive Timeliness Rate | 在规定期限内完成归档的记录占总数的比例。 | 衡量流程执行效率 |
| **数据完整性** | Data Completeness | 核心业务字段的填写齐全程度。 | 评估数据质量 |
| **采购周期** | Procurement Cycle | 从需求书审批完成到结果公示完成的时间周期。 | 衡量采购效率 |
| **合同周期** | Contract Cycle | 从结果公示完成到合同签订的时间周期。 | 衡量签约效率 |
| **周期预警** | Cycle Early Warning | 对即将超期或已超期的业务发出预警提醒。 | 风险防控、流程提醒 |
| **按时完成率** | On-Time Completion Rate | 在规定时间内完成的业务占应完成业务的比例。 | 绩效评估、流程管理 |
| **结算偏差率** | Settlement Variance Rate | `(结算金额 - 合同总额) / 合同总额`，衡量成本控制情况。 | 风险分析、成本管理 |
| **专业报告** | Professional Report | 格式和内容相对固定的周期性报告（周/月/季/年报）。 | 常规工作汇报 |
| **综合报告** | Comprehensive Report | 结构更复杂、分析维度更丰富的深度分析报告。 | 年度总结、部门回顾 |
